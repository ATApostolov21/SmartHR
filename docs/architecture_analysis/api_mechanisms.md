# Механизми на API Заявките (API Mechanisms)

Този документ обяснява как "разговарят" Frontend-ът и Backend-ът, проследявайки пътя на една заявка. Тук са разгледани ключови концепции като CORS, Middleware, валидация и JSON комуникация.

## 1. Структура на API Ендпоинтите

В `api/main.py`, ендпоинтите са дефинирани чрез декоратори на **FastAPI**:

*   **`@app.get("/")`**: HTTP GET метод за четене на данни.
*   **`@app.post("/api/upload")`**: HTTP POST метод за изпращане на данни към сървъра.

Всеки ендпоинт е като "гише" в институция, което отговаря за точно определена услуга.

## 2. Жизнен Цикъл на Една HTTP Заявка (Request Lifecycle)

Представи си, че изпращаш препоръчано писмо (заявката) до институция (сървъра). Ето какво се случва на всяка стъпка:

### Стъпка 1: Клиентът Изпраща Заявка
Когато натиснеш бутон "Analyze" в браузъра, Frontend-ът изпраща HTTP пакет, съдържащ:
*   **Method**: POST (действие: "създай/обработи")
*   **URL**: `http://localhost:8000/api/predict` (адрес)
*   **Headers**: Метаданни (напр. `Content-Type: application/json`)
*   **Body**: Самите данни (JSON с данните на служителя)

### Стъпка 2: Middleware (Портиерът)
Първото нещо, което заявката среща на сървъра, е **Middleware**.
В нашия код (`api/main.py`):
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"], ...
)
```
Middleware-ът е като охраната на входа. Той проверява: "Имаш ли право да влизаш тук?".
*   **CORS (Cross-Origin Resource Sharing)**: Браузърите блокират по подразбиране заявки от един сайт към друг (напр. от React на порт 5173 към Python на порт 8000). Middleware-ът слага печат "Разрешено", за да може браузърът да приеме отговора. Без това, браузърът ще хвърли грешка за сигурност.

### Стъпка 3: Маршрутизация (Routing / Регистратура)
FastAPI поглежда URL адреса (`/api/predict`) и намира коя Python функция отговаря за него (`async def predict_single(...)`).

### Стъпка 4: Валидация на DTO (Data Transfer Objects)
Преди функцията да тръгне, данните минават през скенер – това е **Pydantic**.
Кодът казва: `employee: EmployeeBase`.
Pydantic проверява всяко поле в JSON-а:
*   "Age трябва да е число (int)." – Ако изпратиш текст "двадесет", Pydantic ще спре заявката веднага с грешка `422 Unprocessable Entity` (Невалидни данни).
*   Това гарантира, че функциите работят само с чисти, проверени данни, предотвратявайки сривове по-късно. Тези схеми (`schemas/employee.py`) са DTO – обекти, чиято единствена цел е да пренасят данни.

### Стъпка 5: Изпълнение на Логиката (Controller)
Функцията се изпълнява. Тя извиква `model_service`, прави изчисления и получава резултат (число за риск).

### Стъпка 6: Сериализация на Отговора
Резултатът в Python (напр. числото `0.85`) трябва да стане обратно на текст, който браузърът разбира.
FastAPI автоматично превръща Python речниците и обектите в **JSON формат** (текст). Това се нарича *сериализация*.

## 3. Аутентикация (Authentication)

**В момента: Няма.**
Проектът е проектиран като локален инструмент или отворено демо. Всеки, който има достъп до URL-а, може да прави заявки.
*   **Как би трябвало да бъде в продукция?**
    *   Обикновено се добавя `Authorization` header.
    *   Клиентът праща "токен" (все едно пропуск) с всяка заявка: `Authorization: Bearer <token>`.
    *   Middleware-ът проверява дали токенът е валиден преди да пусне заявката по-нататък.

## 4. CORS Политики (Политики за Споделяне на Ресурси)

```python
allow_origins=["*"]
```
*   **Какво означава?**: Символът `*` означава "всички". Сървърът казва: "Приемам заявки от абсолютно всеки уебсайт".
*   **Риск**: Това е удобно за разработка (за да не настройваш всеки път), но е лошо за сигурността в реална среда, защото злонамерен сайт може да накара браузъра ти да прати данни към твоето приложение.
*   **Правилният подход**: Да се изброят конкретни домейни, напр: `allow_origins=["https://my-hr-app.com"]`.

## 5. Хедъри (Headers) за Кеширане и Сигурност

*   **Кеширане (Caching)**:
    *   В момента не изпращаме специални `Cache-Control` хедъри. Това означава, че браузърът решава сам, но по подразбиране API заявките (особено POST) не се кешират.
    *   Ако искахме да кешираме списъка със служители (който се променя рядко), щяхме да добавим хедър: `Cache-Control: max-age=3600` (помни това за 1 час).
*   **Content-Type**: FastAPI автоматично слага `Content-Type: application/json`, за да каже на браузъра: "Това, което ти пращам, не е просто текст, а структурирани данни".

---
**Обобщение за начинаещи**:
API-то е като сервитьор. Ти (Frontend) поръчваш (Request) по меню (Schema/DTO). Сервитьорът (Controller) носи поръчката в кухнята (Service Layer). Ако поръчаш нещо, което го няма в менюто, сервитьорът ти отказва веднага (Validation Error). Накрая ти носи ястието (Response), опаковано по стандартен начин (JSON).
